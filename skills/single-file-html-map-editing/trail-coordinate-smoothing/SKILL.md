# Trail Coordinate Smoothing

## When to use
- Trail lines on a Leaflet map appear jagged or zigzag, bouncing back and forth instead of flowing along a natural path
- Coordinates were generated by interleaving settlement lat/lng values with original trail points, creating alternating direction reversals
- Consecutive duplicate coordinate points exist (same [lat, lng] repeated)

## Root cause

When replacing straight-line trail coordinates with settlement-based waypoints, a script interleaves settlement coordinates with trail reference points. If the order alternates between "trail position" and "nearest settlement," the result is a bouncing/zigzag pattern:

```
Trail coords BEFORE:   [[49.9, -97.1], [50.1, -97.2], [49.9, -97.3], [50.1, -97.4], ...]
                       ↑ north band      ↑ south band   ↑ back north    ↑ south again

Smoothed AFTER:        [[49.97, -97.14], [49.99, -97.24], [50.01, -97.34], ...]
                       ↑ flows west naturally
```

## Detection

A trail is zigzagging if consecutive latitude differences frequently reverse direction:

```python
# Calculate direction reversals
dlats = [coords[i+1][0] - coords[i][0] for i in range(len(coords)-1)]
reversals = sum(1 for i in range(1, len(dlats)) if dlats[i] * dlats[i-1] < 0)
reversal_ratio = reversals / max(len(dlats) - 1, 1)

# If > 25% of segments reverse direction, it's zigzagging
if reversal_ratio > 0.25:
    print(f"Zigzag detected: {reversal_ratio:.0%} direction reversals")
```

Also check for consecutive duplicates (same point twice in a row):

```python
dupes = sum(1 for i in range(1, len(coords)) if coords[i] == coords[i-1])
```

## Fix approach

### Step 1: Safe extraction using bracket counting (NOT regex)

**DO NOT** use regex to find `coords` blocks — greedy regex like `r'"coords":\s*\[(.*?)\]\n'` will match across multiple trail blocks and corrupt the file.

Use bracket counting instead:

```python
def find_all_coords(text):
    """Find all coords blocks with proper bracket matching."""
    results = []
    search_pos = 0
    while True:
        idx = text.find('"coords":', search_pos)
        if idx == -1:
            break
        
        bracket_start = text.index('[', idx)
        depth = 0
        bracket_end = bracket_start
        
        for pos in range(bracket_start, len(text)):
            ch = text[pos]
            if ch == '[':
                depth += 1
            elif ch == ']':
                depth -= 1
                if depth == 0:
                    bracket_end = pos
                    break
        
        coords_text = text[bracket_start+1:bracket_end]
        results.append((idx, bracket_start, bracket_end, coords_text))
        search_pos = bracket_end + 1
    
    return results
```

### Step 2: Parse coordinates

```python
import re

def extract_coords(text):
    pairs = re.findall(r'\[\s*([\d.\-]+)\s*,\s*([\d.\-]+)\s*\]', text)
    return [[float(lat), float(lng)] for lat, lng in pairs]
```

### Step 3: Smooth with weighted moving average

```python
def smooth_coords(coords):
    # 1. Remove exact duplicates
    unique = [coords[0]]
    for c in coords[1:]:
        if c != unique[-1]:
            unique.append(c)
    
    if len(unique) < 4:
        return coords
    
    # 2. Check zigzag
    dlats = [unique[j+1][0] - unique[j][0] for j in range(len(unique)-1)]
    reversals = sum(1 for j in range(1, len(dlats)) if dlats[j] * dlats[j-1] < 0)
    if reversals / max(len(dlats) - 1, 1) < 0.25:
        return coords  # Not zigzagging
    
    # 3. Weighted moving average: center=0.4, neighbors=0.3 each
    smoothed = [unique[0]]
    for j in range(1, len(unique) - 1):
        lat = 0.3 * unique[j-1][0] + 0.4 * unique[j][0] + 0.3 * unique[j+1][0]
        lng = 0.3 * unique[j-1][1] + 0.4 * unique[j][1] + 0.3 * unique[j+1][1]
        smoothed.append([round(lat, 4), round(lng, 4)])
    smoothed.append(unique[-1])
    
    return smoothed
```

### Step 4: Format and replace

```python
def format_coords_block(coords):
    """Format as compact single-line coords (saves file size)."""
    lines = []
    for i, (lat, lng) in enumerate(coords):
        comma = "," if i < len(coords) - 1 else ""
        lines.append(f"    [{lat}, {lng}]{comma}")
    return "[" + "\n" + "\n".join(lines) + "\n  ]"

# Apply replacements in REVERSE order to preserve string positions
for _, bracket_start, bracket_end, _ in sorted(blocks, key=lambda x: x[0], reverse=True):
```

## Key pitfalls

1. **Regex extraction corrupts the file** — Never use `r'"coords":\s*\[(.*?)\]'` with `re.DOTALL` or `re.findall`. The `.*?` will match across trail boundaries. Always use bracket counting.

2. **Apply replacements in reverse** — When doing string replacements based on character positions, process from end of file to beginning. Otherwise earlier changes shift all subsequent positions.

3. **Preserve first and last points** — The moving average should never modify start/end coordinates, as these anchor the trail. Smoothing only applies to interior points.

4. **Compact format saves space** — After smoothing, you can compress multi-line coordinate arrays (`[\n  [\n    49.89,\n    -97.14],\n  ]`) into single-line (`[49.89, -97.14],`). This reduces file size by ~1200 lines for 45 trails.

5. **Don't smooth trails that aren't zigzagging** — Only smooth if reversal_ratio > 0.25. Some trails already have proper directional flow and smoothing would hurt their accuracy.

6. **Commit BEFORE smoothing** — Always have a clean git commit before running the smoothing script so you can `git reset --hard` if the file gets corrupted.
